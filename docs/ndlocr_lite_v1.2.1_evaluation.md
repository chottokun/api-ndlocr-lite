# ndlocr-lite v1.2.1 調査・整合性評価レポート

## 1. 調査の概要
現在のリポジトリにおける `ndlocr-lite` サブモジュールのバージョンと、最新の `ndlocr-lite` (v1.2.1) を比較調査し、現在のコード (`src/core/engine.py`) との整合性を評価しました。

### 現在の状況
- **サブモジュールバージョン**: v1.1.3 (commit 89f8158)
- **モデル**: 16px 高さの PARSeq モデルを使用 (例: `parseq-ndl-16x...`)
- **実装**: `src/core/engine.py` が `ndlocr-lite` の `ocr.py` をベースに独自に最適化（スレッドプールの再利用、`defusedxml` によるセキュリティ強化など）されています。

## 2. 最新バージョン (v1.2.1) での主な変更点
最新の v1.2.1 では以下の機能追加および改善が行われています。

1.  **認識モデルのアップグレード**:
    - 入力画像の高さが 16px から 24px に変更された新しいモデルが採用されています。
    - 日本語手書き文字や英文タイプライター文字の精度が大幅に改善されています。
2.  **縦中横 (TCY) サポート**:
    - `tcy_wrapper.py` が追加され、新聞などの縦書き中にある横書き文字の認識精度が向上しています。
3.  **カスケード認識ロジックの改善**:
    - 98文字以上の非常に長い行に対して、画像を分割して再認識するヒューリスティックが追加されました。
4.  **出力スキーマの変更**:
    - JSON 出力に `class_index` (認識されたレイアウトのクラスID) が含まれるようになりました。

## 3. 現行コードとの整合性評価

### 肯定的な点 (整合している点)
- **アーキテクチャの共通性**: Detector -> XML生成 -> XY-Cut (読み順) -> Recognizer という基本的なパイプラインは一致しており、現在の `NDLOCREngine` は最新のワークフローを正しく踏襲しています。
- **後方互換性**: XML生成ロジック (`ndl_parser.py`) や読み順ロジック (`eval_xml`) の主要な部分は維持されているため、現在のエンジン実装が動作しなくなるような破壊的変更はありません。

### 懸念点・不一致点 (改善の余地がある点)
- **モデル解像度の乖離**: 最新版は 24px モデルを前提としていますが、現行の `engine.py` のデフォルト値や `Dockerfile` で取得されているモデルは 16px 版です。最新の精度を享受するためには、モデルファイルの差し替えが必要です。
- **機能の欠落**:
    - **TCY 対応**: 現在の API では `TateChuYokoWrapper` が実装されていないため、新聞資料などの特定のレイアウトでの認識精度が最新版に劣ります。
    - **長文分割ロジック**: 98文字以上の行に対するフォールバック処理が `engine.py` には移植されていません。
- **スキーマの不一致**: 最新の `ocr.py` が出力する JSON には `class_index` が含まれていますが、`src/api/main.py` や `src/schemas/ocr.py` はこれを想定していません。
- **XML ライブラリの差異**: 上流の `ndl_parser.py` は `lxml` を使用していますが、本プロジェクトの `engine.py` はセキュリティ（XXE攻撃対策）のために `defusedxml` を使用しています。この方針は維持すべきですが、上流の更新を取り込む際に競合が発生しやすくなっています。

## 4. 結論と推奨事項
現在の実装は `ndlocr-lite` の基本機能を正しく実装しており、整合性は保たれています。しかし、精度向上のための最新モデルへの移行や、TCY サポートといった追加機能の取り込みには、以下の対応が推奨されます。

1.  **モデルの更新**: `parseq-ndl-24x...` 系のモデルへ移行し、環境変数やデフォルトパスを更新する。
2.  **TCY の統合**: `TateChuYokoWrapper` を `engine.py` に組み込み、API オプションとして有効化できるようにする。
3.  **出力スキーマの更新**: JSON 出力に `class_index` を追加し、フロントエンドやクライアントがレイアウト情報を活用できるようにする。
